오늘 한 일

Linux, GNU, kernel, shell에대해 간략하게 배우고 shell 명령어와 git 명령어를 통해 git을 사용해 보았다. github에 계정을 만들고 repository를 생성해 remote에 로컬에 있는 파일들을 올려보기도 하고 remote의 디렉토리를 로컬에 클론해보기도 했다. 또 마크다운 문법으로 md파일과 commit 메세지 등을 작성하고 github 블로그인 hexo 블로그를 만들고 포스팅도 해보았다.

느낀점

강사님이 단순히 git 사용법만 알려주신게 아니라 Git의 배경부터 설명해주신 점이 좋았다. 하지만 아직 모르는 개념이나 용어들이 많아서 수업 내내 나의 정신은 코로나를 뚫고 저 먼 태평양을 건너 미국도 갔다오고 유럽도 갔다오고 아프리카도 갔다왔다. 즐거운 여행이었다.
가끔 카페에 가면 까만화면에 흰글자만 가지고 작업하는 사람들을 보면서 우와 멋지다 했는데 내가 지금 그걸 하고있네? vim은 확실히 익숙하지 않은 환경이라 불편한 점이 아직은 많지만 제대로 익숙해지고나면 마우스로 이동하는 타임로스가 줄어들어 편할 것 같다. 그리고 지금 깨달았는데 한 문장안에서 자동줄바꿈 됐을 시 위아래줄로 커서 이동이 불가하다. 커서가 위아래의 다른 문장으로 가버린다. 이거만 아니면 좀 써볼만 했는데 흠.. 아무튼 자존감이 떨어지는 날이면 한 번씩 사람 많은 카페에 가서 vim으로 작업하거나 git 커밋이라도 한 번 해주면 좋을 것 같다. 화면은 반드시 사람들이 보이는 쪽으로.
초코몽키님의 블로그를 보고나니 매번 공부한 내용을 짧게나마 기록해보자는 마음이 들었다.
한동안 타자연습을 게을리 했더니 영타는 물론이거니와 한타도 너무 느려졌다. 타이핑이 느려서 수업을 놓치지 않도록 내일부터는 무조건 30분씩 타자연습을 해야겠다.

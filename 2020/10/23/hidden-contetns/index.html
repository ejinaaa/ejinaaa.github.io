숨김 콘텐츠 처리 방법

숨김 콘텐츠

  • 마크업은 일종의 섹션과 그 하위 섹션을 구분하는 개념으로 <h1>, <h2>, <h3>, <h4>, <h5>, <h6>를 사용해 섹션의 제목을 정의하는데 이런 섹션의 구획 요소와 제목 요소의 관계가 아웃라인을 형성하게 됨.
    이 때 아웃라인 구조에는 들어가야 하지만 눈으로는 해당 요소를 보이지 않게 하기 위해서 콘텐츠를 숨김 처리함.
    크로스 브라우징과 접근성을 고려해 스크린 리더가 읽을 수 있고 해당 요소의 위치는 그대로 이지만 눈에는 화면에는 보이지 않게 다양한 속성을 넣어 설정함.

숨김 처리 방법

화면에는 보이지 않지만 기계적으로는 접근할 수 있도록 콘텐츠를 숨김 처리함. 크로스 브라우징 될 수 있도록 다양한 스타일을 입혀 숨김.


  • {
  display: none;
  font-size: 0;
}

투명처리를 한 것처럼 자리는 차지하고 눈에는 보이지 않지만 스크린리더와 검색엔진에 정보를 제공하지 못함.

<br>
  • {
  position: abolute;
  top: -9999px;
}

IR기법으로 많이 사용되는 방법이지만 위의 `position: absolute`를 주고 offset을 조정해 사용하는 방법과 동일한 이유로 접근성에서 좋지 않음.
<br>
  • {
  width: 1px;
  height: 1px;
  margin: 0;
}

`clip-path`의 `polygon()`등의 함수를 이용해 픽셀의 path를 0값으로 바꾸어 1px의 너비와 높이를 없앰. 눈에는 보이지 않지만 그 자리에 존재하게 만듬. x, y 순으로 값이 들어가게 됨.  
`clip-path`는 구형 브라우저에서는 적용되지 않을 수 있기 때문에 `clip`속성의 `rect()`함수를 이용해 브라우저마다 최적화 되어 보여지게 함. 이 때 구형 브라우저에서는 컴마를 인식하지 못하므로 컴마가 붙은 `rect()`함수를 밑에 한 번 더 넣어줌. 순서 중요.
<br>
  • {
  opacity: 0;
}

    예전엔 모바일에서 이슈가 있었지만 현재는 가능한 것 같음.

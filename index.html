숨김 콘텐츠 처리 방법

숨김 콘텐츠

  • 마크업은 일종의 섹션과 그 하위 섹션을 구분하는 개념으로 <h1>, <h2>, <h3>, <h4>, <h5>, <h6>를 사용해 섹션의 제목을 정의하는데 이런 섹션의 구획 요소와 제목 요소의 관계가 아웃라인을 형성하게 됨.
    이 때 아웃라인 구조에는 들어가야 하지만 눈으로는 해당 요소를 보이지 않게 하기 위해서 콘텐츠를 숨김 처리함.
    크로스 브라우징과 접근성을 고려해 스크린 리더가 읽을 수 있고 해당 요소의 위치는 그대로 이지만 눈에는 화면에는 보이지 않게 다양한 속성을 넣어 설정함.

숨김 처리 방법

화면에는 보이지 않지만 기계적으로는 접근할 수 있도록 콘텐츠를 숨김 처리함. 크로스 브라우징 될 수 있도록 다양한 스타일을 입혀 숨김.


  • {
  display: none;
  font-size: 0;
}

투명처리를 한 것처럼 자리는 차지하고 눈에는 보이지 않지만 스크린리더와 검색엔진에 정보를 제공하지 못함.

<br>
  • {
  position: abolute;
  top: -9999px;
}

IR기법으로 많이 사용되는 방법이지만 위의 `position: absolute`를 주고 offset을 조정해 사용하는 방법과 동일한 이유로 접근성에서 좋지 않음.
<br>
  • {
  width: 1px;
  height: 1px;
  margin: 0;
}

`clip-path`의 `polygon()`등의 함수를 이용해 픽셀의 path를 0값으로 바꾸어 1px의 너비와 높이를 없앰. 눈에는 보이지 않지만 그 자리에 존재하게 만듬. x, y 순으로 값이 들어가게 됨.  
`clip-path`는 구형 브라우저에서는 적용되지 않을 수 있기 때문에 `clip`속성의 `rect()`함수를 이용해 브라우저마다 최적화 되어 보여지게 함. 이 때 구형 브라우저에서는 컴마를 인식하지 못하므로 컴마가 붙은 `rect()`함수를 밑에 한 번 더 넣어줌. 순서 중요.
<br>
  • {
  opacity: 0;
}

    예전엔 모바일에서 이슈가 있었지만 현재는 가능한 것 같음.

Flow & float BFC

Normal Flow

마크업 된 순서대로 요소들이 자연스럽게 배치되는 흐름을 말한다.

  • nomarl flow에서는 요소가 갖고 있는 기본 display값에 따라 요소들이 배치된다.


  • 모든 요소들은 지정된 width, height값과 상관 없이 눈에 보이지 않지만, 스타일을 통해 자유롭게 이동 또는 배치될 수 있는 영역을 갖게 된다.
    블록 요소는 자신의 콘텐츠의 높이와 부모 요소의 너비만큼의 영역을 갖게 되고, 인라인 요소는 자신의 콘텐츠의 높이와 콘텐츠의 너비만큼의 영역을 갖게 된다.
    이 때 블록 요소가 갖게 되는 영역을 BFC, 인라인 요소가 갖는 영역을 IFC라고 한다.


    이 때문에 블록 요소는 수직으로 떨어져 배치가 되고(BFC의 너비가 부모의 너비만큼 꽉 차게 너비를 갖게 되므로 옆으로 붙을 공간이 없음), 인라인 요소는 (자신이 갖고 있는 콘텐츠 만큼의 너비만 갖고 있으므로 부모영역을 벗어나지 않는 한) 한 라인 안에 수평으로 붙어 배치가 된다.


  • Normal flow에 따라 배치 되는 것은 컴퓨터가 보여주는 방식이고 요소에 css로 특정 속성을 이용해 해당 요소에 새로운 BFC 영역을 생성할 수 있다.


  • BFC가 생성되는 요소 및 상황

    • html 루트
    • float(none제외), position(static제외), overflow(visible제외), display: inline-block속성을 가졌을 때
    • display table값들을 가졌을 때(table요소들)
    • display: flex값을 줬을 때 자식 아이템들
    • display: grid값을 줬을 때 자식 아이템들

float BFC

  • float이 됐을 때 해당 요소가 블록이던 인라인이던 모두 블록 요소로 바뀌게 되고 normal flow를 벗어나 부유되어 새로운 (역시나 부모 너비 만큼의)BFC영역이 생성되며, float속성을 준 요소는 자신의 앞(에 마크업된) 형제 요소에 영향을 받아 배치될 곳이 결정된다.


  • float속성을 준 형제 요소들이 바로 인접해있다면 그들은 새로 생겨난 float BFC영역을 공유하게되어 옆으로 붙어 배치가 되는데, 만약 float속성을 준 요소의 바로 앞(에 마크업된) 형제 요소가 float되지 않았을 경우 상황은 조금 다르다.


    • float된 요소의 앞 형제 요소가 블록 요소(BFC)일 경우
      이 요소는 부모 영역 만큼의 BFC 영역을 갖고 있기 때문에 더이상 들어갈 공간이 없으므로 그 앞 형제 요소의 BFC 영역 밑으로 떨어져 배치가 된다.


    • float된 요소의 앞 형제 요소가 인라인 요소일 경우
      인라인 요소들은 float된 요소들과 같은 BFC를 공유하게 되고, float된 요소들보다 뒷 순서로 밀려나 배치되게 된다.


  • normal flow 안에서는 자신이 갖고 있는 기본 display값에 따라 떨어져 배치될지 옆으로 붙어 배치될지가 결정되는데, float이 된 상태나 형제가 float이 됐을 때는 자신의 앞 요소의 영향을 받아 어떻게 배치될지가 결정된다.


overflow BFC

  • 블록 요소에만 지정할 수 있고, visible값을 제외한 overflow속성의 값이 지정된 요소는 overflow라는 속성이지닌 특성상 자신의 영역을 벗어난 자식들을 컨트롤 해야하기 때문에 일단 자식의 BFC영역을 자기 영역 안에 담으려고 한다. 이에 따라 자식이 float되어 높이를 잃어버린 (height값을 지정되지 않아서)요소에 overflow를 주었을 때 float된 자식의 높이를 모두 담을 수 있게되어 편법으로 많이 사용된다.

BEM (CSS 방법론)

CSS 방법론

CSS 구조를 개선시켜 코드를 읽기 쉽고 효율적으로 만들어 개발 속도를 높이고 디버깅 및 레거시 코드의 새로운 기능 구현을 크게 향상시키기 위해 사용되는 CSS 네이밍 규칙.
OOCSS, SMACSS, BEM 등이 표준으로 많이 사용되고 있음.

BEM(Block Element Modifier)

  • BEM 방법론은 id속성은 사용하지 않고 오직 class만 부여함.
  • 두 개 이상의 단어를 조합할 땐 -(dash)로 이어주고 블록과 요소의 관계를 나타낼 때엔 _(underscore)를 두번 사용해 이어줌.
    예시)
    .block__element‐‐modifier { property: value; }
  • class명을 지을 때의 목적은 그 요소를 반복하여 재사용할 수 있게 하기 위함. 모든 class명은 구체적이고 명료하고 직관적으로 무엇을 나타내는지 누구나 이해할 수 있게 지어주는 것이 좋음.
  • 각 html 태그에는 오직 class명 하나만 사용해야함. 한 태그 안에 두 개 이상 class 사용하지 않음.

Block(전체를 감싸고 있는 블록요소)

블록(block)은 요소들를 담고 있는 컨테이너로 큰 맥락의 구조. header, main, aside, footer등.
블록 요소는 class의 어근을 형성하고 항상 맨 앞에 위치하도록 함.

Element(요소)

요소(Element)는 블록이 포함하고 있는 한 조각. 블록은 전체를 말하고 요소들은 블록 안에 있는 조각들을 말함. 각 요소는 두 개의 _(underscore)로 연결하여 블록 다음에 위치시킴.

Modifiers

블록이나 요소의 외관, 상태를 변화시킬 때 사용. 요소의 스타일이 같다면 다른 영역(다른 블록 안의 요소)이라 할지라도 새로운 클래스를 정의하지 않고 뒤에 -(dash)를 두 개 추가하고 modifier를 표시함. 절대적으로 필요하지 않는 이상, 추가적인 class를 만들거나 똑같은 스타일을 반복하지 않음.
예시) .header__navigation‐‐secondary {…}
